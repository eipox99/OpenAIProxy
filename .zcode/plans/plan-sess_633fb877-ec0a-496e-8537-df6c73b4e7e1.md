## Security Hardening Plan

### Goal
Add authentication + fix weak config before Unraid deployment.

### Approach
Single `AUTH_TOKEN` from `.env` — checked via a FastAPI middleware. No changes needed to individual route files. Login page sets a secure cookie for browser access; API users use `Authorization: Bearer`.

---

### Step 1: `openproxy/config.py` — Add auth setting
Add a single field:
```python
auth_token: str = Field(default="", alias="AUTH_TOKEN")
```
When empty (default), auth is disabled — backward compatible.

### Step 2: `openproxy/utils/auth.py` — NEW file
FastAPI middleware logic lives here as a helper, but the actual middleware is in main.py:
- `verify_auth(token: str) -> bool` — constant-time comparison against `settings.auth_token`
- Returns `True` if `auth_token` is empty (disable auth)

### Step 3: `openproxy/main.py` — Add auth middleware
A single `@app.middleware("http")` that:
- Skips public paths: `/health`, `/login`, `/logout`, `/static/*`
- If `settings.auth_token` is empty, allows all (backward compat)
- Checks `Authorization: Bearer <token>` header first, then `openproxy_token` cookie
- On mismatch: API paths (`/api/*`, `/v1/*`) get 401 JSON; web paths get redirected to `/login`
- Add login/logout routes directly: `GET /login`, `POST /login`, `POST /logout`

### Step 4: `openproxy/web/templates/login.html` — NEW template
Simple centered form with:
- PicoCSS styling matching the existing theme
- Password input field
- Error message slot for invalid password
- "OpenProxy Login" heading

### Step 5: `openproxy/web/templates/base.html` — Add logout
Add a "Logout" link in the sidebar footer section, calls `POST /logout`.

### Step 6: `openproxy/.env.example` — Document new setting
```env
# Authentication: set to a strong random password to enable auth
# Leave empty to disable authentication (default)
AUTH_TOKEN=
```

### Step 7: `openproxy/docker-compose.yml` — Add env var
Add `AUTH_TOKEN` to the environment section.

### Step 8: Fix weak encryption key
Also update `.env.example` to show proper Fernet key generation:
```bash
# Generate with: python3 -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
ENCRYPTION_KEY=
```

### Step 9: Fix `run.sh` — Remove `--reload` flag for production use
Change from `--reload` to proper production flag, or add a comment noting this is for development.

- Any route files (`web/routes.py`, `api/admin.py`, `api/chat.py`, etc.)
- Any frontend JS (`app.js`) — cookies are sent automatically with Same-Origin fetch()
- The proxy logic (`router/proxy.py`)
- Database models or schemas

### Deployment checklist for Unraid
1. Generate a strong `AUTH_TOKEN` (`openssl rand -base64 32`)
2. Generate a real `ENCRYPTION_KEY` (`python3 -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"`)
3. Set both in your `.env` or docker-compose environment
4. (Optional) Add a reverse proxy with HTTPS for internet exposure
5. (Optional) If you want the proxy endpoints working with OpenAI-compatible clients, they just need `Authorization: Bearer <AUTH_TOKEN>` added
